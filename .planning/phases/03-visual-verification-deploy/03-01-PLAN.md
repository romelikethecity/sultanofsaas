  <action>
1. Create a CNAME file in output/ containing `sultanofsaas.com` (required for GitHub Pages custom domain).

2. Create the GitHub repo if it doesn't exist:
   ```
   gh repo create romelikethecity/sultanofsaas --public --source=. --push
   ```
   If the repo already exists, just add the remote and push.

3. Initialize git in the output/ directory (or use a subtree/ghpages approach). The cleanest approach for a static site:
   - Use `gh-pages` branch strategy: create an orphan branch `gh-pages` containing only the output/ contents
   - From the project root:
     ```
     cd output
     git init
     git checkout -b gh-pages
     git add -A
     git commit -m "Deploy sultanofsaas.com"
     git remote add origin https://github.com/romelikethecity/sultanofsaas.git
     git push -u origin gh-pages --force
     ```
   - Note: The main project repo at the root may or may not exist on GitHub. The deploy only needs the output/ content on gh-pages branch.

4. Enable GitHub Pages via the API if not already enabled:
   ```
   gh api repos/romelikethecity/sultanofsaas/pages -X POST -f source.branch=gh-pages -f source.path=/
   ```

5. Set custom domain:
   ```
   gh api repos/romelikethecity/sultanofsaas/pages -X PUT -f cname=sultanofsaas.com
   ```

6. Verify deployment by checking `gh api repos/romelikethecity/sultanofsaas/pages` returns the pages URL.

7. Print the DNS records the user needs to configure (if not already done):
   - CNAME record: `sultanofsaas.com` -> `romelikethecity.github.io`
   - Or A records pointing to GitHub's IPs (185.199.108-111.153)

Note: The site won't be live at the custom domain until DNS is configured. The GitHub Pages URL (romelikethecity.github.io/sultanofsaas) should work immediately.
