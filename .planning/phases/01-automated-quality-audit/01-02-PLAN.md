<task type="auto">
  <name>Task 1: Add SEO checks to audit.py</name>
  <files>scripts/audit.py</files>
  <action>
Read the existing scripts/audit.py (from Plan 01). Add a new `check_seo()` function and wire it into the main orchestrator. Use html.parser from stdlib (NOT regex for HTML parsing).

1. **Build a simple HTML tag extractor** using `html.parser.HTMLParser`:
   - Subclass HTMLParser to extract: title text, meta description content, all `<script type="application/ld+json">` blocks, all `<a href="...">` values, H1 text.
   - This parser runs once per file and collects all needed data in a single pass.

2. **check_seo() function** `check_seo(output_dir) -> List[Finding]`:

   a. **Detect stale/missing output:** If output/ doesn't exist or has fewer than 300 HTML files, return a single info-level Finding: "SEO checks skipped: run build.py first (found {N} pages, expected 334)". Do NOT error.

   b. **Walk output/ directory:** Find all .html files. For each file:

   c. **Unique meta titles (SEO-01):**
      - Extract `<title>` tag content. Error if missing or empty.
      - After scanning all files, check for duplicate titles. Error for each duplicate pair with both file paths.

   d. **Meta descriptions (SEO-01):**
      - Extract `<meta name="description" content="...">`. Error if missing.
      - Warning if length is outside 120-160 characters. Include actual length in message.
      - After scanning all files, check for duplicate descriptions. Error for each duplicate pair.

   e. **BreadcrumbList schema (SEO-02):**
      - Parse all `<script type="application/ld+json">` blocks as JSON.
      - For every page EXCEPT index.html (homepage), check that at least one JSON-LD block has `"@type": "BreadcrumbList"`. Error if missing.

   f. **FAQPage schema (SEO-03):**
      - For pages in the `output/tools/` directory (tool review pages), check if the page content includes an FAQ section (look for FAQ heading or `FAQPage` in JSON-LD).
      - Cross-reference: if the tool has `faqs` in its content dict (from get_tool_content), the page MUST have FAQPage schema. Error if missing.
      - Import TOOLS from build to know which slugs are tool reviews.

   g. **Broken internal links (SEO-04):**
      - Collect all `<a href="...">` values that start with "/" (internal links).
      - For each internal link, resolve to a file path: href="/tools/hubspot/" -> output/tools/hubspot/index.html, href="/best/crm/" -> output/best/crm/index.html, href="/hubspot-vs-salesforce/" -> output/hubspot-vs-salesforce/index.html.
      - Also handle bare paths: "/about/" -> output/about/index.html.
      - Error if the resolved file does not exist in output/.
      - Deduplicate: report each broken target URL once, with count of pages linking to it.

3. **Wire into main:** Add `check_seo(output_dir)` call in the main block after content checks. Add "seo" findings to the combined list. The existing print_report already handles the "seo" category since Finding.category supports it.

4. **Update report summary** to include seo category in the breakdown line.
