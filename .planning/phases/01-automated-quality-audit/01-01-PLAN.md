<task type="auto">
  <name>Task 1: Build audit.py with Finding class, writing checks, and report formatter</name>
  <files>scripts/audit.py</files>
  <action>
Create scripts/audit.py as a single self-contained script. Structure:

1. **Imports and path setup:** Add scripts/ dir to sys.path. Import TOOLS, CATEGORIES, NICHES, INDUSTRIES from build. Import get_tool_content from content. Use only stdlib (os, sys, re, dataclasses, collections).

2. **Finding dataclass:**
```python
@dataclass
class Finding:
    severity: str       # "error" | "warning"
    category: str       # "writing" | "structure" | "data"
    entity: str         # tool slug, niche slug, or industry name
    section: str        # content section name or ""
    rule: str           # short rule ID
    message: str        # human-readable description
    snippet: str = ""   # offending text (first 120 chars)
```

3. **String extraction helper:** Recursive function `extract_strings(obj, path="")` that yields `(path, text)` for every string value in nested dicts/lists. This is what writing checks iterate over.

4. **Writing checks function** `check_writing(tool_slug, content_dict) -> List[Finding]`:
   - **False reframes (AUDIT-01):** Multiple regex patterns, all case-insensitive:
     - `r"(?i)\bnot\s+[\w'-]+[,.]?\s*it'?s\b"` (not X, it's Y)
     - `r"(?i)\bisn'?t\s+[\w'-]+\.\s*It'?s\b"` (isn't X. It's Y)
     - `r"(?i)\bless\s+about\b.*\bmore\s+about\b"` (less about X more about Y)
     - `r"(?i)\bnot\s+so\s+much\b.*\bas\b"` (not so much X as Y)
     - `r"(?i)\bthe\s+pattern\s+isn'?t\b"` (The pattern isn't)
     - `r"(?i)\bstop\s+thinking\s+of\s+it\s+as\b"` (stop thinking of it as)
     - All severity: error

   - **Em-dashes (AUDIT-02):** Search for unicode em-dash character `\u2014`. Severity: error.

   - **Banned words (AUDIT-03):** Case-insensitive word-boundary search for each: robust, leverage, synergy, holistic, cutting-edge, seamless, frictionless, end-to-end, genuinely, truly, really, actually, game-changer, paradigm shift, best-in-class, world-class. Severity: error.
     - IMPORTANT: Exclude tool names from banned word matches. "Seamless.AI" contains "seamless" but is a product name, not a banned usage. Check if the matched text is part of a known tool name before flagging. Use TOOLS dict to get all tool names.

   - **Unearned declarations (warning):** Regex for "The pattern here is clear", "Here's the thing", "The bottom line:", "It's worth noting", "When it comes to", "Without further ado".

   - **Commentary after stats (warning):** "Let that sink in", "You read that right", "That's impressive".

   - **Forced casual (warning):** "Spoiler alert", "Plot twist", "Here's the kicker", "Let's dive in", "At the end of the day".

   - **Tautologies (warning):** "Time will tell", "Success depends on execution".

   - **Missing contractions (warning):** Word-boundary regex for "it is", "do not", "cannot", "will not", "does not", "they are", "we are", "you are" (but not at sentence start for emphasis or in quoted text).

5. **Structure checks function** `check_structure(tool_slug, content_dict) -> List[Finding]`:
   - Required sections (AUDIT-05): overview, expanded_pros, expanded_cons, pricing_detail, who_should_buy, who_should_skip, stage_guidance, alternatives_detail, verdict_long, faqs. Error if missing.
   - Min 3 expanded_pros and 3 expanded_cons, each with "title" and "detail" keys. Error if fewer.
   - Stage guidance: all 4 keys (solo, small_team, mid_market, enterprise). Warning if missing.
   - At least 5 FAQs, each with "question" and "answer". Warning if fewer.
   - At least 2 who_should_buy and 2 who_should_skip. Warning if fewer.
   - At least 3 alternatives_detail. Warning if fewer.
   - Word count (AUDIT-04): Sum all text across all sections. Warning if outside 2,000-4,000 range. Print actual count.

6. **Data checks function** `check_data() -> List[Finding]`:
   - Niche winner uniqueness (AUDIT-06): For each category, find the tool with sultans_pick=True. For each niche, check if rankings[category][0] is the same slug. Error if they match.
   - Industry completeness (AUDIT-07): For each industry, verify picks exist for all 9 category slugs. Error if any missing.
   - Also: verify every tool slug in NICHES rankings exists in TOOLS. Error if not.
   - Also: verify every tool slug in INDUSTRIES picks exists in TOOLS. Error if not.

7. **Report function** `print_report(findings: List[Finding])` (AUDIT-08):
   - Print header: "AUDIT RESULTS: {N} tools scanned"
   - Print summary: "Errors: N (breakdown by category) | Warnings: N (breakdown by category)"
   - Print ERRORS section first, grouped by category (writing, structure, data), then WARNINGS.
   - Each finding: `[{category}] {entity} / {section}: {message}` with snippet on next line if present.
   - Exit with code 1 if any errors found, 0 if only warnings or clean.

8. **Main block:** Orchestrate: load all tools, run writing + structure checks per tool, run data checks, collect all findings, call print_report. Add a `--verbose` flag that also shows tools with zero findings ("CLEAN: hubspot").
